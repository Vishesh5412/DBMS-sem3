1. Introduction & Problem Statement
1.1 Objective
Create a set of SQL views that generate role-appropriate clinical summaries, hiding sensitive
columns based on access level. The system is implemented using core DBMS concepts: SQL
DDL/DML, views, triggers, stored procedures, and rule-based SQL logic only — no machine
learning is required.
1.2 Module Context
Module Number 42
Module Title Secure Clinical Summary View Generator
Category Category G – Secure EHR & Access Control Systems
Key Entities Patient, EHRRecord, UserRole, ClinicalSummary
DBMS Concepts DDL, DML, Constraints, Queries, Triggers, Procedures, Views,
Normalization
1.3 Scope & Limitations
• Scope: 5 core tables with clinical data relevant to Secure EHR & Access Control Systems.
• The system uses SQL-only rule-based logic; no ML or external APIs are required.
• Data used is synthetic/sample clinical data; no real patient PII is included.
• Scope is limited to secure ehr & access control systems as defined by the course
allotment.
